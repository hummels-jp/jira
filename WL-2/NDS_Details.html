NDS 几何表达详述

NDS (Navigation Data Standard) 的道路几何表达核心在于二进制压缩与分层关联。

1. 表达形式与样式

  • 离散化路径：由一系列高精度形状点 (Shape Points) 串联形成道路骨架。
  • 参数化属性：每个 Link 关联宽度、斜度、坡度等属性，而非仅存储点坐标。

2. 逻辑分层示意图

NDS 数据模型分层逻辑
------------------------------------
| Level 1: Topology (路由层) |
| (Node <---> Link 拓扑连接) |
------------------------------------
↓ (ID 关联索引)
------------------------------------
| Level 2: Geometry (物理层) |
| (Compressed Delta Points) |
------------------------------------

3. 示例数据结构说明 (C++ 表现形式)

NDS 底层几何数据采用增量编码 (Delta-encoding) 以实现极致压缩。以下是该数据结构的 C++ 定义示例：

// NDS 二进制几何结构定义示例
struct LinkGeometry {
    uint32_t link_id;
    int32_t start_lat_offset;
    int32_t start_lon_offset;
    uint16_t num_shape_points;
    ShapePoint points[];
};

// 形状点存储：相对于上一点的增量编码
struct ShapePoint {
    int16_t dx; // Delta-X (增量)
    int16_t dy; // Delta-Y (增量)
};
