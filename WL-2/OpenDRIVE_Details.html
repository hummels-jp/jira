OpenDRIVE 几何表达详述

1. 基本概念与 XML 整体结构

基本概念：OpenDRIVE 是一个基于 XML 的道路网络描述标准，核心在于通过参数化参考线定义道路。

2. 几何基元 (Geometry Primitives)

Line (直线)

<geometry length="57.3"><line/></geometry>

Arc (圆弧)

<geometry length="9.23"><arc curvature="-0.127"/></geometry>

Spiral (螺旋线)

<geometry length="1.32"><spiral curvStart="0.0" curvEnd="0.2"/></geometry>

Parametric Cubic Polynomial (参数化三次多项式)

<paramPoly3 aU="0.0" bU="1.0" ... />

3. s/t 坐标系 (s/t Coordinate System)

4. Road Linkage (道路连接)

基础拓扑连接

定义道路间的简单先后继关系，通过 <link> 标签建立前后继道路的索引。

<road rule="RHT" length="100.0" id="1" junction="-1" >
  <link>
    <predecessor elementType="road" elementId="2" />
    <successor elementType="road" elementId="3" />
  </link>
</road>

路口连接 (Junction Linkage)

对于复杂路口，OpenDRIVE 通过 junction 标签来管理多条道路的逻辑交叉与物理过渡。

平滑连接规范 (How to link roads)

工程规范：两条道路的参考线在连接处必须具有相同的航向角 (heading)，且不产生间隙 (gaps) 或 重叠 (overlap)，从而确保路面平滑，维持仿真动力学的准确性。

5. OpenDrive Lane (车道定义)

OpenDRIVE 使用 <lanes> 标签定义道路的车道配置。每一条道路包含至少一个车道段 (laneSection)，并根据参考线进行左右划分。

<lanes>
  <laneSection s="0.0">...</laneSection>
  <laneSection s="15.0">...</laneSection>
</lanes>

关键要素：

  • laneSection: 道路在 s 坐标方向上的分段（如图所示在 s=15.0 处切分），用于描述车道数或类型的变化（如车道合并/增加）。
  • ID 分布: 0 为中心车道（参考线所在），正数表示左侧车道，负数表示右侧车道。

车道段分割示意 (Lane Section Split)

如图所示，在 s = 15.0 处，参考线触发了新的 laneSection，这通常用于描述车道数的变化（例如在该点增加了一条右侧车道）。

车道连接 (Lane Link)

在此基础上，通过 <link> 标签定义车道间的前驱关系 (Predecessor)，从而确保即使在车道数发生变化的情况下，拓扑连接依然保持连续。

6. Junctions (路口)

Junctions 用于定义多条道路汇合的区域。它通过 connection 标签建立道路间复杂的逻辑关系，包含路口的进入道路、离开道路以及具体的车道连接。

<junction id="1" name="Intersection_01">
  <connection id="0" incomingRoad="1" connectingRoad="2" contactPoint="start">
    <laneLink from="-1" to="-1"/>
  </connection>
</junction>

关键要素：

  • connection: 定义了从一条进入道路 (incomingRoad) 到连接道路 (connectingRoad) 的物理路径。
  • laneLink: 定义了在该连接路径中，具体的左右车道是如何进行 ID 映射的，这是实现转向决策的核心。
